# 测试集生成方法对比

## 两种方法对比

### 方法一：基于文档块生成问题（build_testset.py）

从预先分块的文档中生成问题。

**特点：**
- ✅ 每个问题对应一个小的语义单元
- ✅ 标准答案更精确
- ✅ 可以生成更多样化的问题
- ✅ 适合测试具体细节的检索能力
- ❌ 可能丢失全局上下文
- ❌ 问题过于碎片化
- ❌ 难以测试跨文档块的理解

**适用场景：**
- 测试系统对具体信息的检索能力
- 文档已经分块，无需额外处理
- 需要大量小粒度的测试问题

### 方法二：基于完整文档生成问题（build_testset_v2.py）✨ 推荐

从完整的原始文档中生成问题，保持语义完整性。

**特点：**
- ✅ 保持完整的语义上下文
- ✅ 问题更自然、更全面
- ✅ 可以测试需要综合理解的问题
- ✅ 标准答案包含更多上下文信息
- ✅ 更接近真实用户提问场景
- ❌ 长文档可能超出 LLM 上下文窗口
- ❌ 生成的标准答案可能较宽泛

**适用场景：**
- 需要测试系统的综合理解能力
- 文档结构完整，语义连贯
- 追求更真实的评估效果

## 使用建议

### 首次评估：使用方法二（基于文档）

```bash
python rag_eval/build_testset_v2.py \
  --docs-dir "知识库/知识库" \
  --target-questions 50 \
  --questions-per-doc 3
```

### 补充评估：使用方法一（基于文档块）

```bash
python rag_eval/build_testset.py \
  --chunks-file data/all_chunks.json \
  --target-questions 100 \
  --questions-per-chunk 2
```

## 语义完整性对比示例

### 基于文档块的问题

```
文档块: "江西省交通投资集团注册资本100亿元，成立于2015年..."

问题: "江西省交通投资集团的注册资本是多少？"
标准答案: "100亿元"

评价:
- ✓ 精确针对具体信息
- ✓ 标准答案明确
- ✗ 缺乏上下文
- ✗ 问题过于狭窄
```

### 基于完整文档的问题

```
完整文档: "江西省交通投资集团有限责任公司是江西省人民政府批准设立的国有独资公司，
主要负责江西省内交通基础设施的投资、建设和运营管理。公司成立于2015年，注册资本100亿元..."

问题: "江西省交通投资集团的成立背景、主要职责和注册资本是多少？"
标准答案: "江西省交通投资集团是江西省人民政府批准设立的国有独资公司，成立于2015年，
主要负责江西省内交通基础设施的投资、建设和运营管理，注册资本100亿元。"

评价:
- ✓ 保持完整语义
- ✓ 需要综合理解
- ✓ 标准答案丰富
- ✓ 更接近真实提问
```

## 推荐方案

### 综合使用两种方法

1. **主要评估**（70%）：使用基于文档的方法
   - 测试综合检索能力
   - 验证系统对整体内容的理解

2. **补充评估**（30%）：使用基于文档块的方法
   - 测试细节检索能力
   - 验证系统对具体信息的定位

### 实施步骤

```bash
# 步骤 1: 使用基于文档的方法生成主要测试集
python rag_eval/build_testset_v2.py \
  --target-questions 35 \
  --questions-per-doc 3

# 步骤 2: 使用基于文档块的方法生成补充测试集
python rag_eval/build_testset.py \
  --target-questions 15 \
  --questions-per-chunk 2

# 步骤 3: 合并两个测试集并综合评估
python rag_eval/merge_and_evaluate.py \
  --testset1 rag_eval/evals/datasets/ragas_testset_doc_35q.json \
  --testset2 rag_eval/evals/datasets/ragas_testset_15q.json
```

## 总结

| 方面 | 文档块方法 | 文档方法 | 推荐 |
|------|-----------|---------|------|
| 语义完整性 | ⭐⭐ | ⭐⭐⭐⭐⭐ | 文档方法 |
| 问题精确度 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 文档块方法 |
| 真实性 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 文档方法 |
| 实施难度 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 文档块方法 |
| 综合评估 | ⭐⭐ | ⭐⭐⭐⭐⭐ | 文档方法 |

**总体建议：优先使用基于完整文档的方法（build_testset_v2.py），可以获得更准确、更真实的评估结果。**
