> 技术栈: Gradio 4.x
> 日期: 2026-04-30
> 状态: 已确认

---

## 1. 设计定位

### 美学方向：Executive Intelligence Dashboard

企业级战略决策支持平台，传达权威感、信任感和专业性。

| 元素 | 选择 | 理由 |
|------|------|------|
| Typography | 中文：思源宋体(标题) + 思源黑体(正文)；英文：DM Serif Display + Sora | 宋体=庄重，黑体=数据清晰 |
| Color | 主色：深海蓝 `#1a2332`；强调色：琥珀金 `#d4a574`；中性色：暖灰调 | 蓝=权威，琥珀=进度/智慧 |
| Texture | 微妙纸质纹理叠加 | 模拟实体战略报告质感 |
| Motion | 章节切换平滑滑动，状态变化用渐变 | 严肃工具的克制动效 |

### 视觉记忆点

章节里程碑进度条——每个章节是战略路线图上的节点，完成时带琥珀金光晕，传达"战略推进"仪式感。

### 状态视觉规则

| 状态 | 图标 | 颜色 | 效果 |
|------|------|------|------|
| 已完成 | `◉ ✅` | `#4ade80` 翠绿 | 静态，淡灰底线连接下一节点 |
| 进行中 | `◉ 🔵` | `#d4a574` 琥珀金 | 缓慢呼吸光晕动画 (pulse) |
| 待处理 | `○` | `#6b7280` 灰 | 静态，半透明 |
| 已锁定 | `◇` | `#4b5563` 深灰 | 待前序完成解锁 |


## 4. 右侧工作区

### Tab动态解锁机制

Tab按Agent工作进度逐步解锁，避免展示空内容：

| 阶段 | 可见Tab | 说明 |
|------|---------|------|
| Researcher工作中 | 检索文档（实时填充） | 其他Tab灰色锁定 |
| Analyst工作中 | + 分析过程 | 检索文档已填满 |
| Writer工作中 | + 草稿（流式输出） | 三个Tab有内容 |
| LLM评审完成 | + 评审详情 | 审核按钮激活 |
| 发生过修改 | + 修改历史 | 仅修改后出现 |

### Tab 1: 草稿

- Writer生成的章节内容，Markdown渲染
- 流式输出（逐字显示）
- 底部元信息：字数、生成耗时、引用来源数量

### Tab 2: 检索文档

- 每个文档卡片：标题、来源类型、相关度分数、摘要片段
- 默认展示高相关度(>0.85)前3个，其余折叠
- 点击可展开完整内容

### Tab 3: 分析过程

- 完整推理链：问题拆解 → 事实提取 → 推理链 → 洞察输出
- 每个事实标注来源（链接到检索文档）
- 推理链可视化流程展示

### Tab 4: 评审详情

- 独立LLM(Qwen3.5-122B)评审结果
- 分项评分横向进度条：内容准确性/逻辑连贯性/数据支撑/格式规范/战略洞察
- 每个问题：位置定位 + 问题描述 + 改进建议
- 底部操作建议与按钮呼应

### Tab 5: 修改历史

- 按时间线排列，显示操作链路
- 标注每次操作触发的Agent和评审结果
- 历史版本可点击查看完整内容
- 底部提醒单章修改次数限制

## 5. 底部操作区

### 按钮与Agent决策映射

| 按钮 | Agent决策 | 触发路径 |
|------|----------|----------|
| ✅ 批准通过 | `approve` | 进入下一章节 |
| 🔄 补充数据 | `revise:data` | 回到Researcher重新检索 |
| 📊 重新分析 | `revise:logic` | 回到Analyst重新推理 |
| ✍️ 重写内容 | `revise:writing` | 回到Writer重新撰写 |
| ❌ 终止报告 | `finished` | 进入Archiver结束 |

### 交互细节

- LLM评审完成前，按钮灰色不可点击
- 选择修改类操作后展开"补充说明"输入框
- 终止报告需二次确认
- 点击后按钮变loading状态，显示当前执行的Agent名称

## 6. 蓝图审核页（Ch3后特殊节点）

与普通章节的区别：无Tab结构，直接展示结构化内容。

### 展示内容

- **企业使命**: 一句话使命声明
- **SWOT分析矩阵**: 四象限表格（优势/劣势/机会/威胁）
- **TOWS策略组合**: SO/WO/ST/WT策略
- **战略支柱**: 3-5个支柱卡片（主业提质/创新驱动/产业协同/治理升级）
- **关键指标**: BSC框架（财务/客户/运营/学习维度）

### 操作

- ✅ 批准蓝图，进入举措阶段
- 🔄 重新生成蓝图（附修改意见输入框）

---

## 7. 完成页

- 成功提示 + 报告标题
- 统计信息：总字数、生成耗时、引用来源数、章节数
- 操作：查看报告 / 下载报告(.md/.docx/.pdf) / 查看蓝图 / 新建报告

---

## 8. 报告查看页

- 左侧固定目录栏，点击跳转到对应章节
- 正文Markdown渲染，引用标注可点击查看来源
- 下载支持 .md / .docx / .pdf 格式

## 9. 历史报告列表

- 报告卡片：标题、生成时间、状态（已完成/已终止）、章节数、字数
- 操作：查看 / 删除
- 搜索过滤

| 状态变量 | 类型 | 说明 |
|----------|------|------|
| `report_id` | str | 当前报告唯一标识 |

## 11. 实现策略

### Gradio组件选型

| 模块 | Gradio组件 |
|------|-----------|
| 进度面板 | `gr.HTML()` + 自定义CSS |
| Tab工作区 | `gr.Tabs()` + `gr.Tab()` |
| 草稿展示 | `gr.Markdown()` 流式更新 |
| 检索文档 | `gr.HTML()` 卡片列表 |
| 评审分数 | `gr.HTML()` 自定义进度条 |
| 操作按钮 | `gr.Button()` |
| 补充说明 | `gr.Textbox()` |
| 状态管理 | `gr.State()` |
| 报告列表 | `gr.Dataframe()` 或 `gr.HTML()` |

### 后端集成

需要在现有Agent系统中增加：
1. **暂停/恢复机制**: LangGraph workflow在 `human_review` 节点暂停，前端操作后恢复
2. **状态序列化**: 将 scratchpad/context_pool 序列化为前端可渲染的格式
3. **流式输出**: Writer生成内容通过generator流式传输到前端
4. **报告持久化**: 完成报告保存到本地，支持历史列表查询
