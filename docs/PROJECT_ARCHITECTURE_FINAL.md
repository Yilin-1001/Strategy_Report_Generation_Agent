## 模块化设计亮点

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        模块化设计特性                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  1. 单一职责原则                                                             │
│     ├─ data_loader/   仅负责文档处理                                         │
│     ├─ embeddings/    仅负责向量化                                           │
│     ├─ storage/       仅负责向量存储                                         │
│     └─ utils/         仅负责通用工具                                         │
│                                                                              │
│  2. 开闭原则                                                                 │
│     ├─ 通过YAML配置扩展文档类型                                             │
│     ├─ 无需修改代码即可调整参数                                             │
│     └─ 新增Chunker无需改核心逻辑                                            │
│                                                                              │
│  3. 依赖倒置原则                                                             │
│     ├─ Pipeline依赖抽象接口                                                 │
│     ├─ 模块间通过稳定接口通信                                               │
│     └─ 易于替换单个模块                                                    │
│                                                                              │
│  4. 接口隔离原则                                                             │
│     ├─ 每个模块提供清晰的公共接口                                           │
│     ├─ 输入输出类型明确                                                     │
│     └─ 易于测试和维护                                                      │
│                                                                              │
│  5. 配置驱动设计                                                             │
│     ├─ chunking_config.yaml  控制分块策略                                    │
│     ├─ milvus_config.yaml    控制数据库配置                                  │
│     └─ 行为由配置决定，代码无需修改                                         │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```
